# 원본 데이터 폴더 구조

의뢰사가 제공하는 TourAPI 4.0 원본 JSON이 5개 권역별 하위 폴더로 배치되어 있습니다.
공공누리 제3유형(변경금지) 조건에 따라 각 파일의 값은 임의로 수정하지 않습니다.

```
data/raw/
├── 서울/
├── 대전_충청권/   ← 서비스 선정 권역 (backend/.env REGION 값과 일치)
├── 구미_경북권/
├── 광주_전라권/
└── 부산/
    ├── SCHEMA.md               (해당 권역 원본 필드 정의)
    ├── SOURCE.md               (해당 권역 출처·라이선스·파일 목록)
    └── <권역>_<콘텐츠유형>.json  (관광지/레포츠/문화시설/쇼핑/숙박/여행코스/음식점/축제공연행사)
```

## 적재 범위

`backend/app/services/data_loader.py`의 `load_tour_data()`는 **`backend/.env`의 `REGION` 값에
해당하는 폴더 하나만** 읽어 `tour_masters`/`tour_items` 테이블에 적재합니다(의뢰서상 서비스는
5개 권역 중 1곳만 선정해 구축하므로). 기본값은 `대전/충청권` → `data/raw/대전_충청권/`.

```
cd backend
python -m scripts.seed_db
```

다른 권역으로 바꾸려면 `backend/.env`의 `REGION` 값을 해당 폴더명과 같은 형식(`/` 구분,
예: `서울`, `구미/경북권`, `광주/전라권`, `부산`)으로 변경한 뒤 다시 실행하면 됩니다.

각 권역 폴더 안의 `SOURCE.md`가 그 권역 데이터의 출처·라이선스·파일 목록에 대한 1차 자료이며,
`data/SOURCE.md`(선정 권역 요약)도 함께 참고하세요.
